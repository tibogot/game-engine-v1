In the article, a profile is the cross-section at a connection (lanes, widths, types) — not the same word as these Network presets, but each preset here fixes the graph and those cross-sections for you.
Debug fork island: first preset — same fork as the article, but only the merged junction polygon + fork markings are drawn (no road runs, no dead-end caps). Edges stay in the graph so mouths/clips still define the shape; export JSON includes junctionIslandOnly when on.
Article fork (long arms): second preset — three strips on the approach, two on the through leg, one narrow exit; no profiles[] on the approach.
Network mode: click a node to continue, click a segment to select it, Insert knot (split edge) or Alt-click inserts a graph node on that road — this is the practical equivalent of the article’s segment break / place to attach another profile: use it where a branch or exit will meet the corridor, then draw a third road from that node for a junction. Sandbox Spirit’s article inverts names: there, physical pavement pieces are "nodes" and cross-section connectors are "edges"; this lab keeps the usual graph (node = point, edge = road run) and stores cross-sections in laneLayers plus optional profiles[] along each run (see Data Structures of Roads). Split-created nodes export as profileBoundary: true and show a green cross-section tick while the node is degree-2.
Spans, bends, and junction outlines share one mouth clip rule (mouthClipAlongNode) so corners do not leave straight stubs. Dashed guides use the same clipped segment. 3+ legs use merge fill. Roundabout: merged hub, dark island, one dashed circulation ring inboard of mouths, and per-lane spokes into the ring. Road outlines are solid. On multi-strip runs, strip–strip dividers are white dashes (two on a 3-strip highway). At a lane fork, the rightmost divider splits: one Hermite continues onto the through leg; a second branch (solid) follows the ramp outer edge. Widen the approach with edge widthMul (simple preset uses 1.45; through leg 0.967 ≈ ⅔ of that; exit 0.483 ≈ ⅓ for one lane). Profiles along a run: optional per-edge profiles — each { u, laneLayers, widthScale? } with u in [0,1] along the clipped run; the lab interpolates lane stacks and width between keyframes. After a split with no prior profiles, both new runs get explicit u:0/u:1 keyframes so each piece is a clear "segment between profiles". Use preset "Article fork" on a long selected edge, or JSON. Lane fork (article 3→2+1): at a degree-3 node choose junction Lane fork. Put three strips on the incoming highway (laneLayers), two on the through leg, one on the exit ramp; narrow the ramp with edge JSON widthMul (e.g. one-lane ≈ 0.48 when incoming is 1.45 wide). Lane guides: inner strips stay tangent to the straight run; rightmost incoming strip peels to the exit. Use the two buttons to set forkIncomingKey / forkExitKey (else arms are inferred from strip counts at the fork: widest profile = incoming, narrowest = exit; edge length is only a tie-break). Network preset replaces the whole graph with a built-in demo (add more presets in NETWORK_PRESET_LIST in the script).
Optional per-edge headingA/headingB, oneWay, laneLayers in JSON. T-junction mode adds a yield line on the branch arm (exactly 3 roads).
Light edges show piece outlines; green tick = profile knot (profileBoundary).
